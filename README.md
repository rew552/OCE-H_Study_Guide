# OCE-H 試験対策学習ガイド

## 試験概要 / Exam Overview

| 項目 | 内容 |
|------|------|
| **試験名** | Omnissa Horizon Engineer - 2025 (2H0_25) |
| **認定資格** | Omnissa Certified Engineer – Horizon (OCE-H) |
| **問題数** | 65問 |
| **試験時間** | 135分 |
| **合格点** | 285点（スケール方式） |
| **配信** | Pearson VUE（proctored exam） |

### 問題形式 / Question Types

- **Multiple-choice** — 多肢選択（正解1つ）
- **Multiple-selection multiple-choice** — 複数選択（正解が複数）
- **Build-list** — 手順の並び替え
- **Matching** — 用語と説明の組み合わせ
- **Drag-and-drop** — ドラッグ＆ドロップ
- **Point-and-click / Hot-area** — 図上の該当箇所をクリック

### 最小資格要件 / Minimum Qualifications

- 5年以上の一般IT経験、12ヶ月以上の Horizon Suite 経験
- Horizon環境とコンポーネントの高度な管理知識
- Horizon Suite のトラブルシューティング経験
- Workspace ONE UEM との統合経験
- **OCA-H 認定保持が前提**（OCA 試験ブループリントの知識をすべて含む）

## 学習の進め方 / Study Approach

### 推奨学習順序

| 順序 | セクション | 理由 |
|------|-----------|------|
| 1 | **Section 1** — Infrastructure | 環境構築の基盤知識。他セクションの前提となる |
| 2 | **Section 2** — Desktop/App/Profile | デスクトップ・アプリ配信の実務知識 |
| 3 | **Section 5** — Security | SSL/MFA/データ保護は設計・運用に横断的に関わる |
| 4 | **Section 3** — Monitoring & Optimization | プロトコル最適化・イメージ最適化・容量計画 |
| 5 | **Section 4** — Troubleshooting | Section 1〜3 の知識を前提としたトラブル対応 |
| 6 | **Section 6** — Scale and Recovery | CPA・DR/BC は応用的なトピック |
| 7 | **Section 7** — Automation & Integration | API・Access・Intelligence 連携は最終段階 |

### 学習のポイント

- 各ファイルの **重要ポイント** セクションを重点的に確認する
- 設定値・ポート番号・制限値などの数値は正確に覚える
- 「なぜその設定が必要か」というユースケースを理解する
- 可能であればラボ環境で実機操作を経験する

## 用語集 / Glossary

主要用語をカテゴリ別・アルファベット順に整理。

### コアインフラストラクチャ

| 用語 | 説明 |
|------|------|
| **Access Connector** | Omnissa Access と Active Directory 間のディレクトリ同期・認証を担当するコネクタ |
| **Connection Server** | ユーザー認証とデスクトップ/アプリのブローカー機能を提供する中核コンポーネント |
| **Enrollment Server** | True SSO 用の証明書発行を管理するサーバー。Microsoft CA と連携 |
| **Events Database** | Horizon イベント・監査ログを記録する外部データベース（SQL Server / PostgreSQL / Oracle） |
| **Horizon Agent** | VDI/RDSH にインストールされ、プロトコルセッションを確立するエージェント |
| **Horizon Client** | エンドユーザーが Horizon リソースに接続するクライアントソフトウェア |
| **Horizon Edge Gateway** | Horizon 8 環境を Horizon Cloud Service（Control Plane）に接続する仮想アプライアンス。サブスクリプションライセンス管理と Intelligence 連携を担う |
| **Horizon Universal Console** | Horizon Cloud Service 上の統合管理コンソール（サブスクリプション環境用） |
| **Pod** | Connection Server のグループ単位。最大 7 CS、最大 20,000 セッション |
| **Block** | Pod 内のリソース単位。独自の Hypervisor Manager を持つリソースクラスターで構成 |
| **Recording Server** | コンプライアンス目的でユーザーセッションを録画するサーバー |
| **UAG (Unified Access Gateway)** | DMZ に配置され、外部ネットワークからのセキュアアクセスを提供するゲートウェイ |
| **vCenter Server** | VM ライフサイクル管理・プロビジョニングを担う Hypervisor Manager |

### 表示プロトコル・コーデック

| 用語 | 説明 |
|------|------|
| **Adaptive Transport** | ネットワーク状況に応じて TCP/UDP を自動切り替えする Blast のトランスポート機能 |
| **AV1** | AOM 策定のオープンコーデック。最高の帯域幅効率。明示的に有効化が必要 |
| **BEAT (Blast Extreme Adaptive Transport)** | Blast の UDP ベースのトランスポートプロトコル。FEC（前方誤り訂正）によりパケットロス耐性が高い |
| **Blast Protocol** | Horizon のデフォルト表示プロトコル。TCP/UDP 対応。BlastCodec / H.264 / HEVC / AV1 をサポート |
| **BlastCodec** | Omnissa 独自開発のコーデック。CPU ベースのエンコード。テキスト品質に優れる |
| **Blast Optimizer** | 1〜10 の値でユーザーエクスペリエンスとリソース効率のバランスを制御する設定（デフォルト: 5） |
| **Build-to-lossless** | 初期ロッシー表示から段階的にロスレスへ向上するエンコーディング手法。有効化すると GPU エンコードが無効になる |
| **DSCP (Differentiated Services Code Point)** | Blast トラフィックに QoS 優先度を設定するためのマーキング値（0〜63） |
| **H.264** | 標準規格のビデオコーデック。GPU エンコード対応。GPU 搭載環境のデフォルト |
| **HEVC (H.265)** | H.264 の後継コーデック。高圧縮率で WAN 環境に適する。明示的に有効化が必要 |
| **High Color Accuracy (4:4:4)** | H.264/HEVC の YUV 4:4:4 カラースペース設定。グラフィックデザイン・医療画像向け |
| **PCoIP** | Teradici 開発の表示プロトコル。TCP/UDP 4172 を使用 |

### デスクトップ・プール管理

| 用語 | 説明 |
|------|------|
| **Dedicated Assignment** | ユーザーに固定のデスクトップを割り当てる方式。永続デスクトップ向け |
| **Desktop Pool** | 仮想デスクトップをグループ化する論理コンテナ。Floating / Dedicated 割り当てがある |
| **Entitlement** | ユーザー/グループにデスクトップやアプリへのアクセス権を付与する設定 |
| **Floating Assignment** | ログオンごとに任意の空きデスクトップを割り当てる方式。非永続デスクトップ向け |
| **Full Clone** | ゴールデンイメージから完全に独立した VM を作成する方式。永続デスクトップ向け |
| **Golden Image** | デスクトップ/RDSH のベースとなるマスターイメージ |
| **Instant Clone** | 親 VM の実行中メモリとディスクを共有して高速にクローンを作成する技術 |
| **Push Image** | Instant Clone プールのゴールデンイメージを新しいスナップショットに更新する操作 |
| **RDSH Farm** | 複数ユーザーセッションをホストする Windows Server（RDS ホスト）のグループ |

### アプリケーション・プロファイル管理

| 用語 | 説明 |
|------|------|
| **ADMX Template** | Horizon Agent / Client / Blast / PCoIP の GPO 設定用管理テンプレート |
| **App Volumes** | アプリケーションをレイヤー化して VMDK/VHD で配信・管理するソリューション |
| **AppStack** | App Volumes の読み取り専用アプリケーションボリューム |
| **Apps on Demand** | App Volumes と連携し、ユーザーがアプリを起動した時点でパッケージをアタッチする機能 |
| **Condition Sets** | DEM でポリシー適用条件を定義するモジュール |
| **DEM (Dynamic Environment Manager)** | ユーザープロファイル・環境設定を動的に管理するツール |
| **DirectFlex** | DEM の機能。アプリ設定をログイン時ではなくアプリ起動時にロードしてログイン時間を短縮 |
| **FlexEngine** | DEM のクライアント側エージェント。Horizon Agent に統合 |
| **GPO (Group Policy Object)** | Active Directory 経由で Horizon 環境の設定を管理するポリシーオブジェクト |
| **Loopback Processing** | VDI/RDSH の OU でコンピューターの User Configuration を適用する GPO 設定。Replace モードが推奨 |
| **OSOT (OS Optimization Tool)** | ゴールデンイメージの不要サービス無効化・最適化を行う Omnissa 提供ツール |
| **Smart Policies** | DEM の条件付きポリシー。接続元・デバイス・プロトコル等に基づき設定を動的に適用 |
| **Writable Volume** | App Volumes のユーザーデータ保存用読み書きボリューム |

### セキュリティ・認証

| 用語 | 説明 |
|------|------|
| **CRL (Certificate Revocation List)** | 失効した証明書の一覧。Smart Card 認証時の証明書検証に使用 |
| **FIPS 140-2** | 暗号化モジュールのセキュリティ要件を定義する連邦標準規格 |
| **MFA (Multi-Factor Authentication)** | SAML / RADIUS / RSA SecurID / Smart Card 等による多要素認証 |
| **OCSP (Online Certificate Status Protocol)** | 証明書の失効ステータスをリアルタイムに確認するプロトコル |
| **RADIUS** | OTP / Push / SMS 等の二要素認証で使用されるプロトコル。UDP 1812 |
| **RSA SecurID** | ハードウェア/ソフトウェアトークンによる認証。`sdconf.rec` ファイルが必要 |
| **SAML** | SSO を実現するフェデレーション認証プロトコル。Omnissa Access 連携で使用 |
| **SSL Bridging** | LB で TLS を終端し再暗号化して UAG に転送する構成。LB と UAG に同一証明書が必要 |
| **True SSO** | Enrollment Server + Microsoft CA による証明書ベースの SSO。パスワード入力なしでログイン |

### ネットワーク・接続

| 用語 | 説明 |
|------|------|
| **ADAM/LDS** | Connection Server が使用する AD Lightweight Directory Services。Horizon 構成データを格納 |
| **BSG (Blast Secure Gateway)** | UAG 上の Blast プロトコル用セキュアゲートウェイ |
| **CDR (Client Drive Redirection)** | クライアントデバイスのローカルドライブをリモートデスクトップにマップする機能 |
| **Clipboard Redirection** | クライアント-サーバー間のクリップボード共有。方向制御・形式フィルタリングが可能 |
| **JMS (Java Message Service)** | Horizon Agent と Connection Server 間の通信プロトコル。TCP 4001/4002 |
| **MMR (Multimedia Redirection)** | マルチメディアコンテンツをクライアント側でレンダリングする最適化機能 |
| **PSG (PCoIP Secure Gateway)** | UAG 上の PCoIP プロトコル用セキュアゲートウェイ |
| **Quiesce Mode** | UAG のメンテナンスモード。LB に 503 を返し、新規セッションを停止。ローリングアップグレードに使用 |
| **RTAV (Real-Time Audio-Video)** | クライアント側の Web カメラ・マイクをリモートデスクトップで使用する機能 |
| **USB Redirection** | クライアントの USB デバイスをリモートデスクトップにリダイレクトする機能 |
| **VGDS (View Global Data Service)** | CPA における Pod 間のグローバルデータ同期サービス |

### GPU・グラフィックス

| 用語 | 説明 |
|------|------|
| **DirectPath I/O** | 物理 GPU を VM に PCI パススルーする方式。最高性能だが vMotion 不可 |
| **vDGA (Virtual Dedicated Graphics Acceleration)** | 物理 GPU を VM に専用割り当てする方式。vSphere 経由だが vMotion 不可 |
| **vGPU** | 物理 GPU を仮想的に分割して複数 VM で共有する技術（NVIDIA GRID）。vMotion 対応 |
| **vGPU プロファイル** | vGPU の VRAM 割り当てサイズとタイプ（Q: Quadro、B: Virtual PC、A: Virtual Apps、C: Compute） |
| **vSGA (Virtual Shared Graphics Acceleration)** | ESXi の GPU ドライバー経由で API レベルの GPU 共有を提供。追加ライセンス不要 |

### スケール・復旧・ライセンス

| 用語 | 説明 |
|------|------|
| **Boot Storm** | 多数の VM が同時起動することによるリソース競合 |
| **CCU (Concurrent Connected User)** | 同時接続ユーザー数ベースのライセンスカウント方式 |
| **Cloud Pod Architecture (CPA)** | 複数 Pod を Federation として連携させ、グローバルなリソース配信を実現するアーキテクチャ |
| **Global Entitlement** | CPA で複数 Pod にまたがるエンタイトルメント。Scope（ANY / SITE / SITE_AND_ADJACENT）で制御 |
| **GSLB (Global Server Load Balancing)** | マルチサイト構成でユーザーを最適なサイトにルーティングする DNS ベースの負荷分散 |
| **Home Site** | CPA でユーザーの優先接続先として設定するサイト |
| **Logon Storm** | 多数のユーザーが同時ログオンすることによるリソース競合 |
| **Named User (NU)** | ユーザー単位のライセンスカウント方式。1 ユーザーが複数プールを使用しても 1 NU |
| **Pod Federation** | CPA における Pod 間の連携構成 |
