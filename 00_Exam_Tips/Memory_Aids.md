## サイジング

### Connection Server

| CCU | Connection Server数 | 推奨構成 |
|-----|-------------------|---------|
| 0-500 | 1-2台 | 小規模環境 |
| 500-2,000 | 2-3台 | 中規模環境 |
| 2,000-5,000 | 3-5台 | 大規模環境 |
| 5,000以上 | 5台以上 | エンタープライズ環境 |

**計算式**: 必要なConnection Server数 = (CCU / 2,000) + 1（高可用性用）

### UAG

| CCU | UAG数 | 推奨構成 |
|-----|-------|---------|
| 0-1,000 | 1-2台 | 小規模環境 |
| 1,000-5,000 | 2-4台 | 中規模環境 |
| 5,000以上 | 4台以上 | エンタープライズ環境 |

**ベストプラクティス**: UAGとConnection Serverの1:1比率

### RDSH

| 用途 | セッション数/ホスト |
|------|------------------|
| 標準的なオフィス作業 | 50-100セッション |
| 重いアプリケーション | 20-50セッション |

## タイムアウト設定

| 設定項目 | 推奨値 |
|---------|--------|
| アイドルタイムアウト | 30分（1800秒） |
| 最大セッション時間 | 8時間（28800秒） |
| 切断後のログオフ | 10分（600秒） |

### セキュリティ重視環境

| 設定項目 | 推奨値 |
|---------|--------|
| アイドルタイムアウト | 15分（900秒） |
| 最大セッション時間 | 4時間（14400秒） |
| 切断後のログオフ | 5分（300秒） |

## 制限値

### Connection Server

- **最大セッション数**: 2,000 CCU/Connection Server（目安）
- **レプリケーショングループ**: 最大32台

### UAG

- **最大セッション数**: 2,000セッション/UAG（目安）
- **推奨比率**: 1:1でConnection Serverと統合

### Instant Clone

- **プロビジョニング時間**: 通常30-60秒
- **親VM**: 電源オフ状態でスナップショットが必要

### App Volumes

- **AppStackサイズ**: 通常10-50GB
- **Writable Volumeサイズ**: 推奨10-50GB

### DEM

- **プロファイルサイズ**: 推奨100MB以下
- **ファイル共有**: SMB 3.0以上

## アップグレード順序

1. Cloud Pod Orchestrator（CPA環境の場合）
2. Connection Server
3. UAG
4. Horizon Agent
5. App Volumes Agent
6. DEM Agent

## 暗記のコツ

1. **ポート番号**: 主要なポート（443、4001、4172、22443）を優先的に暗記
2. **サイジング**: 計算式を理解して暗記
3. **タイムアウト**: 標準環境とセキュリティ重視環境の違いを理解
4. **パフォーマンス目標値**: 目標値、警告レベル、クリティカルレベルの3段階を暗記
5. **アップグレード順序**: 依存関係を理解して順序を暗記

---

## まとめ

これらの数値、設定値、制限値を暗記することで、OCE-H試験での問題解決が容易になります。特にポート番号、サイジング、タイムアウト設定は頻繁に出題されるため、確実に暗記してください。
