# 5.2 Data Protection

## 概要

データ保護は、Horizon環境のセキュリティとコンプライアンスに重要です。OCE-Hレベルでは、クリップボードとCDRポリシーの設定、USBリダイレクションポリシーの設定が求められます。

## クリップボードとCDRポリシーを切り替える場所とオプションの特定

### クリップボードポリシー

**設定場所**:
- Horizon Console → Policies → Global Policies
- DEM/GPOを使用した設定

**設定オプション**:
- **Disabled**: クリップボードの無効化
- **Enabled**: クリップボードの有効化
- **One-way (Client to Server)**: クライアントからサーバーへのみ
- **One-way (Server to Client)**: サーバーからクライアントへのみ

### CDR (Client Drive Redirection) ポリシー

**設定場所**:
- Horizon Console → Policies → Global Policies
- DEM/GPOを使用した設定

**設定オプション**:
- **Disabled**: CDRの無効化
- **Enabled**: CDRの有効化
- **Read-only**: 読み取り専用

## DEM/GPOを使用したクリップボード/CDRポリシーの設定

### DEMでの設定

**手順**:
1. DEM Management Consoleを開く
2. Policies → New Policy
3. Clipboard/CDR設定を選択
4. ポリシーを適用

### GPOでの設定

**手順**:
1. Group Policy Management Consoleを開く
2. GPOを編集
3. Computer Configuration → Policies → Administrative Templates → VMware → VMware Horizon Agent
4. Clipboard/CDR設定を有効化

> [!IMPORTANT]
> DEM/GPOを使用することで、ポリシーを一元管理できます。

## USBリダイレクションポリシーの設定

### USBリダイレクションの設定

**設定場所**:
- Horizon Console → Policies → Global Policies
- DEM/GPOを使用した設定

**設定オプション**:
- **Disabled**: USBリダイレクションの無効化
- **Enabled**: USBリダイレクションの有効化
- **Whitelist**: 許可リスト
- **Blacklist**: ブロックリスト

### デバイスタイプの制御

**設定項目**:
- ストレージデバイス
- プリンター
- スマートカードリーダー
- その他のデバイス

> [!TIP]
> USBリダイレクションポリシーを適切に設定することで、セキュリティと利便性のバランスを取ることができます。

## クリップボード監査

### 監査の設定

**監査項目**:
- クリップボード操作の記録
- データ転送の記録
- ユーザー情報の記録
- タイムスタンプの記録

**監査ログの確認**:
```powershell
# クリップボード監査ログの確認
Get-Content "C:\ProgramData\VMware\VDM\logs\vdm-security.log" | 
    Select-String "Clipboard" | 
    Select-Object -Last 50
```

## USBデバイスの詳細管理

### デバイスタイプの詳細制御

**制御可能なデバイスタイプ**:
- **ストレージデバイス**: USBフラッシュドライブ、外部HDD
- **プリンター**: USBプリンター
- **スマートカードリーダー**: スマートカードリーダー
- **その他のデバイス**: カメラ、オーディオデバイスなど

**ホワイトリスト/ブラックリストの設定**:
```powershell
# GPOでのUSBデバイス制御
# Computer Configuration → Policies → Administrative Templates → 
# System → Removable Storage Access
```

## データ漏洩防止（DLP）の統合

### DLP統合の概要

**DLPソリューション**:
- サードパーティDLPソリューションとの統合
- データ転送の監視
- データ漏洩の防止

**統合方法**:
- APIベースの統合
- エンドポイントエージェントとの統合
- ネットワークベースの統合

## ベストプラクティス

1. **ポリシー管理**
   - 明確なポリシー定義
   - 定期的な見直し
   - ユーザーへの通知

2. **セキュリティ**
   - 最小権限の原則
   - 定期的な監査
   - 監査ログの分析
   - DLP統合の検討

3. **コンプライアンス**
   - 規制要件への対応
   - 監査ログの保持
   - レポートの生成

## まとめ

データ保護の適切な設定は、Horizon環境のセキュリティとコンプライアンスに重要です。クリップボード、CDR、USBリダイレクションポリシーの適切な設定、クリップボード監査、USBデバイスの詳細管理、DLP統合により、セキュアなHorizon環境を構築できます。
